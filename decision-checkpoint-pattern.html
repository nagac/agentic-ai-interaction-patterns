Decision Checkpoint

Before the Publishing Agent runs, the user reviews every product and explicitly approves, rejects, or requests an edit. Nothing goes live without a human decision.

00 — Design Principle
Why this pattern exists
"Publishing is permanent. The user must own this moment — not the agent."
The Decision Checkpoint is the most important pattern in the agentic library. Every other pattern builds trust gradually. This one is where trust is either confirmed or broken. If AI publishes something the user didn't approve, no amount of good design elsewhere will recover that trust.
✓
Full live preview
User sees exactly what will appear on the live page — not a summary, not a thumbnail. The real thing.
⚖️
Three actions, not two
Approve, Reject, or Request edit. Binary approve/reject forces bad decisions. Edit requests keep agents in the loop.
⚡
Bulk when confident
After reviewing a few, users gain confidence. Bulk approval for high-QA items respects their time without removing control.

03 — Rules
Do's and Don'ts
The most important rules in the entire library. This is the last line of human control.
✅ Always do this
✓Always show a full live preview — never a thumbnail or summary
✓Always offer all three actions: Approve, Reject, Request edit
✓Always show QA score on every product so user can calibrate trust
✓Always show "PREVIEW — Not live yet" label on the preview panel
✓Always show a count confirmation before bulk approve executes
✓Always allow undo until the final publish confirmation
✓Always route edit requests back to only the relevant agent
❌ Never do this
✗Never publish without explicit user approval — not even one product
✗Never pre-select Approve — user must make an active choice
✗Never hide the Reject option — it must always be equally visible
✗Never auto-approve high QA scores — QA is advisory, not decisive
✗Never execute bulk approval without the count confirmation step
✗Never send edit request without telling user which agent will re-run
✗Never skip this pattern for "efficiency" — it is non-negotiable
