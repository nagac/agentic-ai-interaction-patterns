Instruction Input

00 — Why This Is Agentic
A vague instruction becomes a wrong agent run at scale
In standard UX, a vague search query returns imperfect results. The user tries again. In agentic systems, a vague instruction triggers 4 agents across 100 products autonomously. The cost of ambiguity is enormous. The instruction input must prevent that before anything starts.
Standard UX — low stakes ambiguity
User types "show me red shoes" in a search box. Results are imperfect. User refines. Cost: 10 seconds.
Agentic — high stakes ambiguity
User types "update all products" without specifying what. 4 agents run on 100 products. Wrong outputs across the entire catalogue. Cost: hours of recovery work.
The agentic moment in this pattern
After the user submits their instruction — before Pre-flight Plan loads — AI reads the instruction and confirms it understood correctly. If something is unclear, it flags it now. Not after agents have already started running.
01 — The Input Screen
Two things. Scope and instruction. Nothing else.
Scope — which products agents will run on, filtered by collection, category, or tag. Instruction — what agents should do. Parameters, constraints, and agent sequence all live in Pre-flight Plan. Not here.

Anatomy — four elements
1 — Header
AI identity and clear framing. User knows they're setting up an agent run — not just typing a search.
2 — Scope selector
Filter by Collection, Category, or Tag. Selected count shown immediately. User always knows exactly how many products agents will touch.
3 — Instruction field
Free text. Hint chips for common additions — constraint, tone, deadline. Nothing more. Parameters live in Pre-flight Plan.
4 — Submit
Shows total task estimate before submitting. Submit button labels the next step — "AI will check" — so user knows what happens next.

04 — Rules
Do's and Don'ts
✅ Always do this
✓Always show product count when scope is selected — user must know exactly what agents will touch
✓Always run clarity check after submit — before Pre-flight loads
✓Always show what AI understood — not just whether it understood
✓Always give a concrete example when instruction is unclear — never just "try again"
✓Always ask one clarifying question at a time — never multiple
✓Always show total task estimate before submit — scope × agents
❌ Never do this
✗Never skip clarity check — even for returning users with familiar instructions
✗Never ask for parameters here — they belong in Pre-flight Plan
✗Never allow submit with zero products selected
✗Never show a generic "I didn't understand" — always say specifically what's missing
✗Never auto-proceed past a flagged instruction — user must confirm
✗Never use this screen to collect agent configuration — keep it to scope and intent only
