Interruption & Pause

The user decides to stop agents mid-run. Not because something broke. The user chose to intervene. The system stops cleanly, preserves all work, and holds state indefinitely until the user is ready.

00 — Why This Is Agentic
The user is taking back control of an autonomous system
In standard UX the user is always in control — nothing to interrupt. In agentic systems agents run autonomously. The user has to actively reach in and stop them. That requires a designed safe stop mechanism.
The agentic problem
Agents running on 10 products simultaneously. User spots something wrong — wrong parameter, changed brief. Needs to stop everything without losing completed work.
The design challenge
Stop agents cleanly — not mid-output. Preserve all completed work. Hold state exactly. Give user full visibility of where everything paused.
The design decision
Agents finish their current product task then pause. Mid-task stop corrupts output. Clean stop preserves every completed product and resumes from the next pending one.

02 — Finishing State
Between pause clicked and fully paused
User clicked pause. Agents are finishing their current tasks. Can last 10–60 seconds. User must know the system heard them and is stopping cleanly.

Why this state exists
User clicked pause but agents don't stop instantly. Each is mid-task. Cut immediately and output is corrupted. This state tells the user: we heard you, we're stopping cleanly, here's which agents are still finishing.
⚠ Design rule
Never show a generic spinner. Always show which specific agents are finishing vs already paused.

The three options
▶ Resume
Agents pick up from next pending task. No repeated work. Unchanged parameters unless user edited them.
Edit → Resume with updates
User changes a parameter. Button becomes "Resume with updates". Updated parameters apply only to remaining products — completed products are never re-run.
✕ Cancel run
All agent work stops permanently. Completed outputs are still preserved — nothing deleted. User taken to summary of completed work.
04 — Rules
Do's and Don'ts
✅ Always do this
✓Always show pause button while agents are running — never hidden
✓Always require confirmation before stopping — prevents accidental pauses
✓Always finish current task before pausing — never cut mid-output
✓Always show the finishing state — user must know the system heard them
✓Always show a snapshot of exactly where everything paused
✓Always preserve completed work — even if run is cancelled
❌ Never do this
✗Never cut agents mid-output — corrupts work and breaks trust
✗Never auto-resume — user must explicitly choose to resume
✗Never delete completed outputs when run is cancelled
✗Never show a generic spinner during finishing state
✗Never re-run completed products when resuming with updated parameters
✗Never hide the pause button inside a dropdown or menu
