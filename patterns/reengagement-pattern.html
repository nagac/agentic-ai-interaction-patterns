Re-engagement

The user left while agents were running. They come back — minutes or hours later. Agents have finished, failed, or changed state. The user needs to re-orient instantly — what happened, what needs them, what's done.

00 — Why This Is Agentic
The user was absent while the system acted autonomously
In standard UX the user is always present — no concept of coming back. In agentic systems agents run without the user watching. When the user returns, the world has changed without them. Re-engagement is the designed re-entry point.
The re-engagement problem — unique to agentic systems
🕐
Time has passed. The user has no mental model of what changed. Gone 5 minutes or 5 hours.
🤖
Agents acted on their behalf without them watching. Some succeeded. Some failed. Some are waiting.
⚡
User needs to re-orient instantly — what needs me right now — without reading through everything.
The design principle
Re-engagement is not a dashboard. It's a briefing. Like a good assistant telling you what happened while you were out — most important first, full detail on demand, clear on what needs you and what doesn't.

Priority order — always fixed
1st — Needs you now
Stuck agents and failures. Something is blocked. Always shown first, never below good news.
2nd — Awaiting approval
Products ready to publish. Agents done. User sign-off needed before next step.
3rd — What happened
Timeline of events. Most recent first. Context without requiring user to read everything.
4th — Completed
Everything that finished successfully. Shown last — no action needed. But user wants to know.

03 — Rules
Do's and Don'ts
✅ Always do this
✓Always show how long the user was away — gives temporal context
✓Always show issues first — never bury them below good news
✓Always show the 4-number summary strip — instant orientation
✓Always show what changed in chronological order — newest first
✓Always provide a direct action CTA — never just information
✓Always use green treatment when everything went well — don't understate good outcomes
❌ Never do this
✗Never show completed items before issues — good news first is wrong here
✗Never dump a full log — timeline shows key events only
✗Never require the user to navigate elsewhere to see what happened
✗Never show same treatment regardless of outcome — always match variant to result
✗Never omit the away time — "while you were away" without context is useless
✗Never show re-engagement card if agents still running and nothing needs user — use Variant C
