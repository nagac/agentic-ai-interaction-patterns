Decision Checkpoint

An agent pauses at a pre-declared irreversible boundary and waits for human sign-off before crossing it. Not a failure. Not an error. A designed trust boundary — agreed in Pre-flight Plan, honoured mid-run.

00 — Why This Is Agentic
A designed trust boundary in an autonomous chain
The agent didn't fail. It didn't get confused. It reached a point it was always told it would reach — and stopped exactly as agreed. This is the agentic contract being honoured. The tone is not alarming. It's a handover.

P-007 Decision Checkpoint

The contract
In Pre-flight Plan, the user saw: "Publishing Agent — will pause for your review before publishing anything live." That was the agreement. Decision Checkpoint is that agreement being kept. The agent ran autonomously — then stopped at exactly the right moment.

What happens to each decision
✓ Approved
Publishing Agent publishes immediately. Product goes live. Appears in Completion Summary.
↻ Request edit
Agent fixes the specific note. Product returns to Decision Checkpoint for re-review. No full re-run — only the flagged element is fixed.
✕ Rejected
Agent output discarded. Product not published. Logged in Completion Summary as rejected. User can start a new run for this product separately.

05 — Rules
Do's and Don'ts
✅ Always do this
✓Always show the checkpoint as a planned, positive moment — not an alert or error
✓Always show all products at a glance before requiring drill-in
✓Always show QA score per product — user must be able to make informed decisions
✓Always provide navigation arrows in drill-in — user must move through all products without returning to overview
✓Always fix only the specific element on request edit — never re-run entire agent chain
✓Always reference the Pre-flight Plan agreement — "as agreed before you started"
❌ Never do this
✗Never style this like an error — teal not red. Agent succeeded.
✗Never allow Publishing Agent to bypass this checkpoint — even for returning users
✗Never make bulk approve the default — user must consciously choose it
✗Never re-run all agents when user requests an edit — only fix what was flagged
✗Never publish anything without explicit user approval on this screen
✗Never show this pattern if Publishing Agent was not declared in Pre-flight Plan
