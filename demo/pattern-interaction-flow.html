P-000 Instruction Input Full Page
How do you help the user write an instruction that agents can act on safely at scale?
Full page because the user navigated here intentionally. Not interrupting anything. Needs space for scope selector + instruction field + clarity check.
👤 User navigates here
→ On submit: Clarity check inline → then Modal P-001
Submit + clarity check passes
↓
P-001 Pre-flight Plan Modal
How do you tell the user what agents will do before they start?
Modal because it's triggered immediately after submit. User is still on the instruction page. Blocking — must approve before anything starts. Not a new destination.
⚙ System — after clarity check
→ On approve: Modal closes → Navigate to Full Page P-002
→ On cancel: Modal closes → Back to P-000
User approves
↓
The hub — everything connects here
P-002 Item Status Grid ★ Central hub Full Page
How do you show progress across all products and all agents while running?
Full page because it's the main monitoring screen. User stays here while agents run. All other patterns branch from here and return here. This is home base.
👤 User navigates here 🤖 After Pre-flight approval 🕐 On return after being away
★ Three patterns branch off this page: P-003 (drawer), P-004 (inline), P-005 (modal). All return here when resolved.
Three things can happen while agents are running
P-003 Graceful Handoff Drawer
How do you pass control back to the human when an agent reaches its limit?
Drawer — slides in from the right over the Status Grid. User can still see which agents are running behind it. Context preserved. Closes when decided.
🤖 Agent reaches decision limit
→ On decide: Drawer closes → Back to P-002
P-004 Error Recovery Inline
How do you recover when an agent fails unexpectedly mid-chain?
Inline — the failed product row in the Status Grid expands in place. No overlay, no navigation. User sees the error in context of the full run. For multiple failures → Drawer.
⚙ Agent fails unexpectedly
→ On fix: Row collapses → Back to P-002 updated
→ Multiple failures: → Drawer instead
P-005 Interruption & Pause Modal
How do you let the user stop agents mid-run without losing work?
Modal because the user triggered it intentionally via the Pause button. Blocking — they need to decide: resume or cancel. Small focused decision. Doesn't need full page.
👤 User clicks Pause button
→ On resume: Modal closes → Back to P-002
→ On cancel: Modal closes → Summary screen
Publishing Agent hits boundary declared in P-001
↓
P-007 Decision Checkpoint Side Panel
How do you pause an agent at an irreversible boundary and get human sign-off?
Side panel — product list on the left, full preview detail on the right. Both visible simultaneously. Faster to review 10 products than navigating in and out of a full page. User never loses their place in the list.
🤖 Publishing Agent hits pre-declared boundary
→ All approved: Navigate to Completion Summary
→ Request edit: Agent fixes → returns to P-007
→ Mixed: Continue reviewing remaining products
If user left and came back at any point
P-006 Re-engagement Banner / Full Page
How do you re-orient a user who returns after agents ran without them?
Banner if user returns to a screen mid-session — thin strip at top with summary + CTA. Full page if it's the first thing they see on opening the app. Match to entry context.
🕐 User returns after being away
→ CTA takes user to most urgent pattern
→ Banner: dismisses after user acts
End of run
— Completion Summary Full Page
How do you close the loop on a completed agent run?
Full page. The run is over. User deserves a clear summary of what was completed, approved, rejected, and published. Starting point for the next run.
⚙ All products decided at P-007 👤 User cancels run at P-005
Container Types — When to Use Each
The decision rule for every pattern in your library.
Full Page
User came here intentionally
Replaces current screen. User navigated here on purpose. Needs full space. Not interrupting anything.
P-000 P-002 P-006* Completion
Use when: user has a clear destination in mind and needs maximum space to work.
Modal
Blocking decision needed now
Overlay on current screen. Blocks everything behind it. User must decide before continuing. Small to medium content.
P-001 P-005
Use when: user triggered this intentionally or system needs a decision before anything else can happen.
Drawer
Context must stay visible
Slides in from side. Current screen visible behind. User keeps context of what's running while handling the interruption.
P-003 P-004*
Use when: agent interrupted the user mid-run and user needs to see the rest of the run while deciding.
Inline
Single item needs attention
Content expands in place. No overlay. No navigation. The failed item row expands with full error detail. Everything else stays visible.
P-004 single
Use when: one specific item has an error. Multiple errors → switch to Drawer instead.
Side Panel
List + detail simultaneously
Two panes visible at once. List on left, detail on right. User moves through items without losing their place in the list.
P-007
Use when: user needs to review multiple items sequentially and compare detail against the full list.
Banner
Non-blocking awareness
Thin strip at top of screen. User can see it and dismiss or act. Doesn't block current work. Used for passive notifications.
P-006 mid-session
Use when: user is already in the app and agents finished while they were on another screen.
