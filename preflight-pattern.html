04 — Rules
Do's and Don'ts
Non-negotiable standards every designer must follow when using this pattern.
✅ Always do this
✓Show Pre-flight Plan before any multi-agent task runs — no exceptions
✓Mark every irreversible step with ⚠ warning badge before user approves
✓Show Cancel in the middle — always accessible, never hidden
✓Use time ranges — "4–6 min" not "5 minutes exactly"
✓Write agent descriptions in plain language the user understands
✓Use Warning variant when any irreversible action is in the plan
❌ Never do this
✗Skip this screen for "experienced" users — familiarity doesn't remove risk
✗Auto-start agents without showing this plan first
✗Hide the publishing step in small text or collapsed section
✗Use technical agent names like "pipeline_executor_v2"
✗Show a precise time you can't guarantee — breaks trust immediately
✗Disable Cancel at any point — user must always be able to stop
