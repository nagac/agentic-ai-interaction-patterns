Graceful Handoff

When an agent reaches its limit mid-chain, it passes control back to the human with full context — what it found, why it stopped, exactly what it needs — then resumes exactly where it left off.

Pattern IDP-003
ModeAgent

00 — Why This Is Agentic
This pattern only exists because an agent is acting autonomously
Standard UX handles human review flows. This pattern handles what happens when a machine reaches its own decision limit mid-chain and needs to bring a human back in without breaking the flow.
The agentic problem
Agent is running autonomously. It hits a borderline case. It cannot decide alone. It has no way to ask. Without this pattern — it either fails silently or makes a wrong call.
The design challenge
Re-engage the user mid-chain. Give them enough to decide quickly. Don't lose the chain state. Resume seamlessly. All without making the user feel the agent failed.
The pattern solves
Agent pauses cleanly. Hands human exactly what they need. Human decides in context. Agent resumes from the exact same point. Chain continues. No lost work, no broken state.

1
Header
Which agent stopped, on which item, at which stage. Never just "Agent needs input" — always specific.
→ Answers: "Where am I in the process?"
2
Context — What the agent found
Three lines maximum. Product, threshold, finding. Enough to understand the situation. No noise.
→ Answers: "What happened?"
3
Evidence
Only the scores or data directly relevant to the decision. Never full audit logs here. Link to full detail if needed.
→ Answers: "How bad is it?"
4
The Question
One clear, specific question in plain language. Never vague. Never multiple questions in one.
→ Answers: "What exactly do you need from me?"
5
Agent Recommendation
The agent's best guess at the right answer, clearly labelled as a recommendation not a decision. Always optional for user to override.
→ Answers: "What would you do?"
6
Pause Status
Shows what the other agents are doing while user decides. Matches the preference they set in Pre-flight Plan.
→ Answers: "Is everything waiting for me?"

04 — Resume State
After the user decides — agent resumes seamlessly
The hardest part of Graceful Handoff to design. The user makes a decision and the agent picks up exactly where it left off. The UI must confirm this clearly.
User decides → Agent resumes
✓
Decision received — resuming
QA Agent picking up from Air Max Lite
✓Your decision: Approve Air Max Lite as-is
✓QA Agent resuming at product 7 of 10
✓Chain state preserved — no work lost
⏳Continuing with products 8, 9, 10...

This confirmation shows for 3–5 seconds then transitions back to Item Status Grid. It is never skipped — the user must always know the agent resumed.

The resume state must always confirm three things
1. What the user decided
Show the decision back to them. This is both confirmation and an implicit audit log. User must be able to see what they approved.
2. Where the agent is resuming from
Exactly which item, which stage, which position in the chain. "Product 7 of 10" not just "resuming".
3. That no work was lost
Explicitly state chain state is preserved. Users fear the agent will start over. Reassure them directly.

⚠ Never combine with
Auto-resolution — the system must never silently decide for the user when an agent reaches its limit. The handoff is the point. Bypassing it defeats the purpose entirely.
06 — Rules
Do's and Don'ts
Non-negotiable. These rules protect user trust in autonomous systems.
✅ Always do this
✓Always name the specific agent, product, and stage in the header
✓Always ask one specific question — never multiple at once
✓Always show the pause status — user must know what other agents are doing
✓Always show the resume confirmation — never silently continue
✓Always preserve chain state — user should never lose work when they decide
✓Always label the recommendation as a recommendation — never as a decision
❌ Never do this
✗Never auto-resolve — agent must always wait for the human
✗Never show a generic "Agent failed" message — always explain specifically
✗Never dump full logs or audit trails in the handoff card — link to detail
✗Never use Variant A (borderline) style for a hard block — severity must match visuals
✗Never restart the agent from the beginning after user decides
✗Never skip the resume confirmation — the user must always know the agent picked back up
07 — Metrics
How you know this pattern is working
Graceful Handoff is working when users make fast, confident decisions and trust the agent to resume correctly.
Time to decide
<45s
From handoff appearing to user making a decision. Longer means the handoff isn't giving enough clarity.
🚨 Red flag if over 3 min
User doesn't understand what's being asked
Override rate
<30%
How often users ignore the agent recommendation and choose differently.
🚨 Red flag if over 60%
Agent recommendations are not calibrated to user values
Handoff frequency
Watch
